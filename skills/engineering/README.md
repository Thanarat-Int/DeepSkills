# Engineering skills

The build lifecycle, idea → running in production. Seven senior/principal-level disciplines.

- **[spec](./spec/SKILL.md)** — Decide what to build and in what order, then lock it into a buildable spec with testable acceptance criteria (prioritization + requirements).
- **[architect](./architect/SKILL.md)** — Design system structure, data model, and security together, recorded as a lightweight ADR.
- **[build](./build/SKILL.md)** — Build it right: scaffold a new repo, implement in test-first verified increments, choose test levels, refactor safely; never "done" without running it.
- **[scrutinize](./scrutinize/SKILL.md)** — Outsider-perspective end-to-end review of a plan, PR, or code change.
- **[ship](./ship/SKILL.md)** — Get to production and keep it healthy: deploy/infra/observability, the readiness gate, deliberate rollout, and release notes.
- **[operate](./operate/SKILL.md)** — Keep running software healthy: diagnose bugs, hunt performance bottlenecks, and write the post-mortem.
- **[handoff](./handoff/SKILL.md)** — Keep project state in durable files so a fresh chat can resume; checkpoint during work, resume ritual on a new session.

> Specialist clusters live in their own buckets: AI/ML in [`ai/`](../ai/) (data-engineering, ml-modeling, mlops, llm-engineering) and games in [`game/`](../game/) (game-design, game-narrative, game-dev).
